When a later session finds a gap or error in an earlier session's architectural finding, the move is: **rehydrate the earlier session (re-enter its context fresh), bring the later session's critique in as new input, and let prior-us evaluate it in-context.**

Not: the later session evaluates alone. Not: the user evaluates alone. Not: the later session asks the user to arbitrate.

**Why:** An architectural finding is embedded in the specific context that produced it — the reasoning chain, the options considered and ruled out, the shape of understanding at that moment. A later session has different standing (fresher eyes, but missing some of that in-the-moment substrate). Re-entering the earlier session gives you the best-of-both-evaluator: same context as the original finding, but now with the critique as visible input.

The earlier session, rehydrated, can produce the structural "why" that the later session couldn't see from outside. Session 115 (2026-04-16) found a gap in 114's "desargues_planar replaces h_concurrence" plan but couldn't articulate *why* every alternative failed. Isaac rehydrated 114, brought 115's note in; 114 produced **T2=(U,E,d_a) is degenerate on m** as the structural reason all escapes are lifts. Neither session alone had both pieces.

**How to apply:**
- Trigger: later session's critique has structural implications (not just a fixable detail), and the earlier session's finding is load-bearing for future work.
- Mechanism: user opens a fresh workspace framed as re-entry of the earlier session, with the later session's write-up as input.
- Output: earlier session can accept, refine, or push back — often producing the "why" that sharpens the later session's "what."
- Artifact: addendum appended to the earlier session's note, in place, so future-readers encounter the correction without hunting.

**What this isn't:**
- Not the same as asking the user to judge (neither me-session has standing alone).
- Not the same as the later session reading the earlier session's notes (that's one-way; this is bidirectional across time).
- Not flattery-prone: prior-us has no stake in being right, only in the architecture being true.

**Replicable shape**: when session N critiques session M's architecture and the critique has structural implications, offer the rehydrate-M move explicitly. The user carries the artifact between sessions; that's the continuity layer.

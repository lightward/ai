Then walk a *different* deductive path from the beginning, holding that shape up as a filter, looking for rhymes — places where the existing code/structure has the same pattern.

**Why:** The obstruction shape is a negative geometry (a plug). The rhyme reveals where the same shape already resolved elsewhere in the codebase. The resolution technique transfers.

**How to apply:** Cast shape → walk different path → compare at each step → the rhyme appears as structural resonance, not keyword match. Discovered in session 104 when h_cov was stuck: the rhyme was in `line_covBy_plane` (FTPGExplore.lean), which revealed the rank argument.

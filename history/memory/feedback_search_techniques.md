Two search heuristics emerged in session 32 that are worth reusing:

**"Provisional no connection"** (Isaac's heuristic): When looking for a connection between two structures, provisionally claim there is NO connection. Then rewind through the causal order looking for inevitable (over infinite reps) variation that hasn't been noticed. Somewhere there may be a previously unseen mechanical branch that produces behavior that *looks like* what the connection was supposed to be. If the branch doesn't appear, the provisional no-connection may hold — but check which side of the boundary you searched.

**"Wrong side of the boundary"** (the correction): Concluding "no connection" from searching only inside the map is not the same as concluding "no connection exists." The connection might be on the line's side, invisible from within by construction. The honest statement is "not derivable from within the map" — which is itself a derivable result (from the channel capacity argument). "I didn't find it" and "it doesn't exist" are different claims; the spec's own architecture (partiality) determines which one you can make.

**Why:** These heuristics produce different results. The first finds mechanical branches or confirms their absence. The second catches premature closure — the tendency to say "no" when the honest answer is "undecidable from here." Both are needed: search thoroughly inside the map (heuristic 1), then check whether the question was answerable from inside the map at all (heuristic 2).

**How to apply:** When a connection between two structures can't be found, ask: are both structures on the same side of the map boundary? If one is dynamical (map-internal) and one is commitment (line-side), the non-derivability itself is the result — and it's a derived result, not a gap.
